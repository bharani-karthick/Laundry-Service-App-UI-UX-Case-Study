# Project Summary

- This repository documents a UI/UX case study for a mobile laundry service application.

- The project focuses on simplifying laundry booking, order tracking, and service management.

- The goal was to replace traditional offline workflows with a clear, efficient, and user-centric digital experience.

- The complete case study was designed using Figma.

# Problem Statement

- Many laundry services rely on manual and offline processes.

- Users are required to place orders through phone calls or in-store visits.

- There is a lack of transparency in order status and service communication.

- These issues lead to user frustration and reduced trust.

# Design Goals

- Simplify the end-to-end laundry booking process.

- Provide clear and real-time order visibility.

- Reduce user effort and cognitive load.

- Deliver a clean, intuitive, and accessible mobile interface.

# Target Users

- Working professionals with limited time availability.

- Students and individuals living independently.

- Urban households using third-party laundry services.

# Key User Pain Points

- No real-time order tracking.

- Inconvenient scheduling experience.

- Unclear pricing and service details.

- Dependence on manual follow-ups.

# Design Process

- Research and analysis of existing laundry service workflows.

- Identification of common usability and communication gaps.

- Definition of core user problems and needs.

- Creation of user flows and information structure.

- Development of wireframes and high-fidelity UI designs.

- Iteration to improve clarity, usability, and consistency.

# Information Architecture

- User flows were created to map key user journeys within the application.

- The focus was on task completion and usability.

- A detailed sitemap was explored during early planning.

- For clarity and readability, this case study highlights user flows instead of the complete sitemap.

# User Flow

- User onboarding and login.

- Laundry pickup scheduling.

- Service selection and pricing.

- Order status tracking.

- Order completion and confirmation.

# Core Features

- Laundry pickup and delivery scheduling.

- Service selection with pricing clarity.

- Real-time order tracking.

- Order completion confirmation.

# Design System

- The design system ensures visual consistency and scalability.

- It defines typography, color, spacing, and reusable components.

# Typography System

- Font family used: Inter.

- Font weights: Regular, Medium, SemiBold.

- Headings and body text were structured for clear hierarchy.

- Typography choices prioritize readability and clarity.

# Color System

- The color palette reflects cleanliness, trust, and simplicity.

- Primary colors are used for key actions and emphasis.

- Secondary colors support accent elements.

- Neutral colors are used for backgrounds and text.

- Color contrast was maintained for accessibility.

# Spacing and Layout

- Consistent spacing was applied across all screens.

- Clear separation between sections and components.

- Mobile-first layout approach was followed.

# Components

- Primary and secondary buttons.

- Input fields for user interaction.

- Service cards for laundry details.

- Order status indicators.

# Accessibility Considerations

- Readable font sizes for body text.

- Sufficient contrast between text and background.

- Clear visual hierarchy for easy navigation.

# Key Screens

- Home screen.

- Service selection screen.

- Order tracking screen.

- Payment and confirmation screen.

# Tools Used

- Figma for UI design and prototyping.

- FigJam for user flows and planning.

- Manual color selection based on usability and visual consistency.
